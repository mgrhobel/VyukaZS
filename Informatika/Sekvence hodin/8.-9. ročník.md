#### H9 – Algoritmy: rekurze a složitost
```python
# Faktoriál rekurzivně
def faktorial(n):
    if n <= 1:
        return 1
    return n * faktorial(n - 1)

# Fibonacciho posloupnost
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# Proč je to pomalé? Vizualizace stromu volání
# Optimalizace: memoization

# Binární vyhledávání
def binarni_hledani(seznam, hledany):
    levo, pravo = 0, len(seznam) - 1
    while levo <= pravo:
        stred = (levo + pravo) // 2
        if seznam[stred] == hledany:
            return stred
        elif seznam[stred] < hledany:
            levo = stred + 1
        else:
            pravo = stred - 1
    return -1
```

#### H13 – Databáze: úvod a pojmy
```
Proč databáze?
  Excel: 1 tabulka, 1 uživatel, malé objemy
  Databáze: relace, integrita, více uživatelů, velké objemy

Pojmy:
  Tabulka (entita): Žáci, Předměty, Učitelé
  Atribut (sloupec): jméno, datum_narozeni, trida
  Záznam (řádek): jeden žák
  Primární klíč (PK): unikátní identifikátor (id_zaka)
  Cizí klíč (FK): odkaz na jiný řádek (id_ucitele v tabulce Předměty)
```

#### H14 – Návrh databáze: ER diagram
```
ER diagram: Entity-Relationship

Entita: obdélník s názvem
Atributy: elipsy u entity
Vztah: kosočtverec mezi entitami, kardinalita (1:N, M:N)

Příklad: Školní databáze
  Žák (id, jméno, třída)
  Třída (id, označení, třídní_učitel)
  Předmět (id, název, učitel_id)
  Hodnocení (id, žák_id, předmět_id, známka, datum)

Aktivita: Navrhni ER diagram pro "Půjčovnu knih"
```

#### H17 – SQL: JOIN (spojování tabulek)
```sql
-- Vypis jméno žáka a jeho průměrnou známku
SELECT z.jmeno, AVG(h.znamka) as prumer
FROM zaci z
JOIN hodnoceni h ON z.id = h.zak_id
GROUP BY z.id, z.jmeno
ORDER BY prumer;

-- Kteří žáci nemají žádné hodnocení?
SELECT z.jmeno FROM zaci z
LEFT JOIN hodnoceni h ON z.id = h.zak_id
WHERE h.id IS NULL;
```
