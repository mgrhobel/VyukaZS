Cviceni SSPGM 12. tyden

1. Analyzujte program p32.c nasledovne:
- dany program dle viz dale modifikujte, prelozte a nastavte
pro nej bit propujceni identifikace
- takto vykonavatelny soubor umistete do sdileneho adresare, ktery
zpristupnite jako "read" vasim kolegum
- ve stejnem adresari vytvorte soubor napr. stejneho jmena jako login_name
uzivatele a tomuto souboru dejte pouze pravo read pro vlastnika souboru
- analyzujte proces, ktery pres mechanismus propujceni identifikace (realne
ID uzivatele, efektivni ID uzivatele) zpristupni soubor jineho uzivatele,
ktery "normalne" uzivateli neni pristupny (viz vyse)
- uvedte priklady, kde se tento mechanismus pouziva

2. Napiste skript pro vyhledavani zadaneho souboru ve zvolene stromove
strukture. Jako pozicni parametry zadejte:
1. parametr - pocatecni hloubka prohledavani(vnoreni) - zadejte 0
2. parametr - nazev hledaneho souboru
3. parametr - adresar zacatku prohledavani - zadejte /home/tmp

Pozadavky:
1. Otestujte pocet a spravne zadani pozicnich parametru
2. Po nalezeni souboru bude nasledovat vzdy vypis:
Hledany soubor "" nalezen v adresari "" v hloubce ""
Hloubkou se mysli adresarove vnoreni od /home/tmp , hloubka = 0
3. Zaroven vypiste obsah hledaneho souboru
4. Pouzijte rekurzivni volani daneho skriptu:
napr. bash /HOME/bin/p17.sh $j $2

Poznamka:
V prohledavane adresarove strukture se nachazi vice souboru
se stejnym nazvem a jinym obsahem.
