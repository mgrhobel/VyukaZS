1.1 Programovací jazyk Java

Java je poměrně nový, ale silný programovací jazyk. Tento jazyk má
ohromný dopad z mnoha

důvodů.

Prvním důvodem je jeho nezávislost na platformě, což znamená, že
aplikace napsaná pro jeden

počítač velmi pravděpodobně poběží nezměněna i na jiném počítači.
Programátor tedy bude moci

napsat jedinou aplikaci, kterou lze spustit na různých typech počítačů.

Java je objektově orientovaný jazyk, což je jeho další výhodou.
Objektově orientované jazyky

zapouzdřují data a metody pro jejich změnu do oddělených jednotek
(objektů). Objekty usnadňují

vytváření a udržování velkých programů.

Další výhodou Javy je její relativní jednoduchost. Samotný jazyk Java má
syntaxi jednodušší než

jazyk C a C++ (z něhož vychází). Jako příklad může sloužit přidělování a
uvolňování paměti v jazyce

C, kde se často dělají chyby. V Javě se přidělování a uvolňování paměti
děje automaticky.

Jediné co v Javě působí obtíže je používání ohromujícího počtu různých
tříd, které přicházejí

spolu s Javou v jejím aplikačním programovacím rozhraní API (Application
Programming Interface).

1.2 Objekty

Objektově orientované programování (OOP) je procesem, během něhož se
podle vlastností a

chování reálných objektů vytvoří software odrážející tyto vlastnosti a
chování. Reálné objekty jsou

objekty z reálného světa - auto, domy, stromy, trojúhelník, čtverec..

Libovolný reálný objekt je charakterizován svými vlastnostmi a svým
chováním. Jako vzor pro

vytvoření uvedeme automobil. Automobil má určité vlastnosti (barvu,
rychlost, cíl cesty, spotřebu) a

určité chování (startování, zastavení, otáčení kol atd.).

Ve světě software rozumíme objektem takovou komponentu softwaru, jejíž
struktura se podobá

struktuře objektu v reálném světě. Každý objekt se skládá z kombinace
dat a chování. Vlastnosti jsou

proměnnými, které určují základní charakteristiky objektu. Metody pak
popisují, jak se objekt chová a

jak může být modifikován. Objekt je tedy prvek software, který se skládá
z proměnných a s nimi

spojených metod.

Softwarový objekt můžeme znázornit asi takto:

Obrázek 1: Obecný popis objektu

Jádro objektu je tvořeno proměnnými. Vnější prostor kolem jádra vyplňují
metody objektu. Jádro s

daty je před vnějším světem ukryto za vrstvou metod.

Jen metody objektu mohou manipulovat s daty objektu. Říkáme, že proměnné
objektu jsou

zapouzdřeny v objektu.

Proměnné a metody objektu se v Javě nazývají proměnné instance a metody
instance, abychom je

odlišily od proměnných třídy a metod třídy (o nich budeme mluvit
později).

Zapouzdření

Používá se k zatajení podrobností o implementaci objektu jiným objektům
v programu. Jestliže ostatní

objekty programu nemohou vidět vnitřní stav objektu, nemohou ani při
náhodných modifikacích stavu

objektu do něj zanést chyby. Navíc změny týkající se vnitřních operací
objektu nebudou mít vliv na

operaci určenou pro ostatní objekty v programu.

Zapouzdření poskytuje dvě základní výhody. A to:

·Modularitu

·Zatajení informací

Modularita: objekt lze napsat a udržovat nezávisle na zdrojovém kódu
ostatních objektů.Takto může

být objekt velmi snadno znovu používán a předáván v rámci systému.

Zatajení informací: proměnné instance objektu nejsou přístupné ostatním
objektům přímo. Z tohoto

důvodu můžeme proměnné a metody objektu změnit, aniž by přitom tyto
změny jakkoliv ovlivnily

ostatní objekty, které jsou na něm nezávislé.

1.3 Zprávy

Objekty si posíláním zpráv říkají co je třeba udělat. Chceme-li objekt o
něco požádat, odešleme

mu zprávu. Zprávy můžeme považovat za požadavky na zavolání metody.
Reakce objektu na zprávu

může být jednoduchá odpověď objektu (vrácení hodnoty, změna jeho
vnitřního stavu, odeslání zprávy

jinému objektu, vytvoření nového objektu nebo kombinace uvedených
možností .

V objektu prostřednictvím proměnných tedy uchováváme data (stav
objektu). Pomocí zpráv

(volání metod) po něm můžeme chtít, aby na sobě vykonával operace, které
mění tyto data.

V podkapitole Objekty byl obecný popis objektu. Nyní si zvolíme
konkrétní objekt jménem ucet1

na němž popíšeme jeho metody a proměnné.

Objekt ucet1 s jeho daty a metodami můžeme po vzoru předchozího schématu
znázornit asi takto:

Obrázek 2: Objekt ucet1 s jeho daty a metodami

Máme zde objekt jménem ucet1. Tento objekt má tři proměnné instance
jménem

vlozenaCastka, vybiranaCastka, zustatek. A tři metody instance jménem
vlozeni(), vyber(),

zjistiZustatek().

Pokud budeme chtít vložit do pokladny určitou částku, pošleme objektu
zprávu(zavolá se metoda

vlozeni()), pomocí které vložíme do pokladny peníze. Chceme-li zjistit
zůstatek, pošleme objektu

zprávu (zavolá se metoda zustatek()), která vypíše hodnotu proměnné
zustatek.

Pozn.: Konkrétní práci s metodami a proměnnými instance vysvětlíme v
kapitole Objekty.

1.4 Třídy

Jsou softwarové předpisy, podle kterých se vytvářejí objekty. Třída je
softwarový pojem, který

určuje počet a typ proměnných instance, které jsou součástí objektu, a
metody instance, které se na

tento objekt budou aplikovat.

Třída je tedy vzorem pro vytváření objektů., sama objektem není. Třída
nám říká, jak bude objekt

vypadat a jak se bude chovat, jakmile bude vytvořen. Podle tohoto
předpisu daného třídou se vytvoří

v paměti každý objekt. Ze stejné třídy lze navíc inicializovat mnoho
různých objektů.

1.5 Balíky

Skupiny příbuzných tříd Javy se sdružují do knihoven, které nazýváme
balíky. Aplikační

programovací rozhraní Javy (API), zahrnuje mnoho balíků, které
implementují důležité vlastnosti

tohoto jazyka. V kapitole Balíky si ukážeme používání API rozhraní a
uvedeme některé nejdůležitější

balíky, které jsou při programování nejfrekventovanější.

Shrnutí některých nejdůležitějších faktů.

·Ideálně by měla být Java jazykem nezávislým na platformě.

·Java je objektově orientovaný jazyk.

·Objektem rozumíme samostatnou softwarovou komponentu, která se skládá z

vlastností(proměnných) a metod.

·S objekty komunikujeme prostřednictvím zpráv.

·Třídou rozumíme softwarové předpisy, podle kterých se vytváří objekty.

·Skupiny příbuzných tříd Javy se sdružují do knihoven, které nazýváme
balíky.
